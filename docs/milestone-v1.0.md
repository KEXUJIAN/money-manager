# Milestone v1.0: 核心功能全面完成

> 本里程碑合并并整理了 v0.1 ~ v0.6 的所有迭代成果，标志着网页版与 PC（Tauri）端的主要功能已全面落地。

## 一、核心架构

| 技术栈 | 选择 | 备注 |
| --- | --- | --- |
| 前端框架 | React + Vite | SPA 架构 |
| 桌面端 | Tauri | Rust 驱动, 轻量打包 |
| 样式系统 | Tailwind CSS 4 + Shadcn/UI | CSS 变量主题 |
| 数据存储 | Dexie.js (IndexedDB) | 本地零服务端 |
| 状态管理 | useLiveQuery | Dexie 原生响应式 |
| 路由 | React Router | 路由懒加载 |
| 动画 | Framer Motion | 物理级动效 |
| 构建优化 | manualChunks + React.lazy | 分包 + 按需加载 |

## 二、功能模块总览

### 2.1 账户管理
- 账户 CRUD（渐变卡片、总资产概览）
- 账户类型支持（现金、信用卡等）
- 余额偏置 (`balanceOffset`) 无痕平账机制
- 单账户数据级联清除（含转账孤岛清理）

### 2.2 记账功能
- 支出 / 收入 / 转账 三种类型
- 转账采用双边模式：`accountId`（出金） + `transferToAccountId`（入金）
- 时间选择器精确到分钟，秒数归零存储
- 备注限制 50 字符，为空自动填充分类名
- 账户 / 分类下拉支持搜索，自动填充默认值
- 快捷新建分类（记账面板内联弹窗）
- 流水编辑与删除（Dexie 原子事务，先撤销原余额影响再应用新变更）

### 2.3 统计分析
- 月度概览、每日趋势折线图
- 分类饼图（尾部 <2.5% 自动聚合为"其他"，支持下钻展开）
- 饼图色彩基于黄金角度 HSL 生成，无限分类不重色
- 时间维度：日 / 周 / 月 / 年 / 全部（差异化选择器）
- 转账流水不计入收支统计
- 明细表格：全局搜索、多维筛选（类型 / 账户 / 时间）、分页

### 2.4 数据管理
- TXT 导入 / 导出（SOH 分隔符格式，升序排列）
- JSON 导入 / 导出（全量备份恢复）
- XLSX 导出（精确数值类型，杜绝科学计数法）
- XLSX 导入（外键强映射，覆盖式解析）
- 导入防重比对，弹窗交互选择保留/跳过
- 清空数据时保留内置分类，全屏 Loading 遮罩
- TXT 导入智能预检：无账户自动创建，有存量弹窗选择

### 2.5 设置页面
- 分类管理：折叠面板、搜索过滤、同类型防重
- 全局消除原生 `alert`/`confirm`，统一自定义弹窗 + Toast
- Toast 文案全中文化

### 2.6 布局与响应式
- PC 端：侧边栏导航
- 移动端：底部 Tab 栏 + 顶部 Header
- 全站中文化

### 2.7 开发者工具
- `/dev-db` 隐藏路由：Dexie 底层数据全量审视
- `@tanstack/react-virtual` 虚拟滚动优化海量记录渲染

## 三、关键技术决策

| 决策项 | 方案 | 理由 |
| --- | --- | --- |
| 状态管理 | useLiveQuery | Dexie 原生响应式，避免额外依赖 |
| 浮点精度 | big.js | 彻底消除金额计算漂移 |
| 主键策略 | Date.now() 序贯字符串 | 零依赖、有序、支持导入去重 |
| 动画引擎 | framer-motion | Shadcn 无法实现滑动跟手物理动画 |
| XLSX 处理 | xlsx (SheetJS) | 原生 CSV 无法精确控制 Excel 类型 |
| 分类初始化 | bulkPut 幂等写入 | 防止重复初始化 ConstraintError |
| 余额平账 | balanceOffset 偏置 | 不注入虚假流水，数学维度纯净 |
| 转账模型 | 双边账户字段 | 出入金清晰可追溯 |
| 构建优化 | manualChunks + React.lazy | 主 chunk 从 1227kB↓367kB (↓70%) |

## 四、已知限制

- IndexedDB 存储受浏览器配额限制（通常 > 1GB）
- 暗色模式 CSS 变量已定义但未完整适配
- Recharts 与 React 19 存在部分类型不兼容，使用 `any` 绕过
